# 构建与发布

Methanol 输出的是纯静态站点，因此 `dist/` 可以部署到任何静态托管平台。

## 构建

```bash
npx methanol build
```

将会生成构建好的 `dist/` 目录。

## 预览

```bash
npx methanol serve
```

用于在本地检查生产输出。

## 部署

将 `dist/` 上传到你选择的托管平台（GitHub Pages、Netlify、Cloudflare Pages、S3 等）。请注意，平台需要能够自动解析不含 `.html` 的 URL 到对应的 HTML 文件。

## 子路径部署

如果站点部署在子路径下（例如 `https://example.com/docs/`），请在 `methanol.config.*` 中设置 `site.base`：

```js
export default () => ({
	site: {
		base: '/docs/'
	}
})
```

dev（`methanol dev`）模式下 `site.base` 会被忽略并当作 `/`。请用 `methanol build` + `methanol serve` 验证子路径部署效果。
