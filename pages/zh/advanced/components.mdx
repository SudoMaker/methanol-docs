# 自定义组件

Methanol 会加载 `components/` 目录中的组件，并让这些组件在所有 MDX 页面里默认可用。它们由 [rEFui](https://github.com/SudoMaker/rEFui) 渲染，所以可以直接使用 rEFui 的 JSX 运行时与 API。

## 组件发现规则

- 支持 `.jsx` / `.tsx` / `.js` / `.ts`
- 文件名即组件名
- 同名时，用户组件会覆盖主题组件

示例：

```
components/
  Callout.jsx
  Hero.client.jsx
  Toc.static.jsx
```

## 客户端组件

当组件赖浏览器 API 时，使用 `.client` 后缀。

```
components/ThemeSearch.client.jsx
```

## 静态组件

需要在构建时求值时，使用 `.static` 后缀。若客户端组件无对应服务端组件时，Methanol 将会尝试使用客户端组件静态渲染内容。如果客户端组件使用了服务端不存在的 API，则必须提供一个对应的静态组件以供服务端渲染。

```
components/Notice.static.jsx
```

## 引入与使用

```jsx
// components/Callout.jsx
export default function Callout({ title }, ...children) {
	return (
		<div class="callout">
			<strong>{title}</strong>
			<div>{...children}</div>
		</div>
	)
}
```

```jsx
<Callout title="Tip">
  段落尽量短一些，方便快速浏览。
</Callout>
```
