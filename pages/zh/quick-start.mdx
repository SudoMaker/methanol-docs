# 快速上手

Methanol 可以把一个装有 Markdown/MDX 的目录直接变成带路由、导航与构建流程的静态站点。

几乎不需要额外配置：创建 `pages/index.mdx`，再逐步添加文件/目录，路由与导航会自动生成和更新。

## 快速运行

想快速体验，直接使用 `npx` 运行：

```bash
npx methanol dev
```

然后打开 `http://localhost:5173`。

## 使用已有的 Markdown 文件夹

如果你已经有一份 Markdown/MDX 文档目录，可以直接指向此目录：

```bash
npx methanol dev ./my-docs
```

该目录会作为内容根目录，如 `./my-docs/index.mdx` 将会对应 `/`。

## 加速本地开发

如果使用频繁，建议全局安装（启动更快）：

```bash
npm install methanol -g
methanol dev
```

## 生产构建

```bash
npx methanol build
npx methanol serve
```

`serve` 会基于 `dist/` 的产物进行本地预览。

## 项目结构

```
pages/        # MDX/MD 页面（或者 docs/）
components/   # MDX 中可用的 JSX/TSX 组件
public/       # 静态资源（覆盖主题的 public 资源）
build/        # 中间 HTML（可选）
dist/         # 生产输出
```

`public/` 中的文件会在 `/` 下提供（例如 `public/logo.png` → `/logo.png`），并会覆盖主题提供的 public 资源。

## 创建第一个页面

创建 `pages/index.mdx`：

```mdx
---
title: 首页
---

# 欢迎

这是我的站点。
```

文件路径会自动转换为路由：

```
pages/index.mdx       -> /
pages/guide.mdx       -> /guide
pages/blog/post.mdx   -> /blog/post
```
