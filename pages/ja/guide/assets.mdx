# 画像とアセット

静的アセットは `public/` に置き、ページ固有のファイルは相対パスで参照します。

## Public フォルダ

`public/` の内容はサイトのルートから配信されます。

サイトをサブパス配信する場合（`site.base !== '/'`）でも、通常は public アセットに `ctx.withBase()` を付ける必要はありません（Methanol がビルド時に解決します）。

ただし JS（client-only コンポーネント）で URL を生成していて Vite の dev で解決が不安定な場合、`ctx.withBase('/...')` が回避策として役立つことがあります。

```
public/
  logo.png
  downloads/
    brochure.pdf
```

MDX から使う例:

```md
![Logo](/logo.png)
[Download the brochure](/downloads/brochure.pdf)
```

## ローカルファイル

ページの横に置いた画像を相対パスで参照できます。

```
pages/
  guide/
    writing.mdx
    writing-diagram.png
```

```md
![Diagram](./writing-diagram.png)
```

## グローバルスタイルとスクリプト

`pages/style.css` を置くと全ページに読み込まれます。`pages/index.js` または `pages/index.ts` を置くとモジュールスクリプトとして読み込まれます。

本番ビルドでは base を考慮した URL で注入されます（`ctx.withBase('/style.css')` や `ctx.withBase('/index.js')` 相当）。

```html
<link rel="stylesheet" href="/style.css" />
<script type="module" src="/index.js"></script>
```
